Object

  • 객체란 사물이다. * 명사와 동사이다
  •  객체=변수(프로퍼티) + 함수(메서드)
  •  객체 선언
  •  객체 리터럴( {} )을 이용한 객체 생성
  •  생성자 함수를 이용한 객체 생성
  • 자바스크립트에서 객체란?
  •  상태(=프러퍼티, 필드)와 동작(=메서드)의 묶어서 저장하는 컨테이너. 캡슐화
  •  명사 => 상태 => 변수 => 프로퍼티
  •  객체의 상태를 나타내기 위한 정보.
  •  프로퍼티(property) = '이름:값' 의 쌍 객체에서 선언된 변수
  •  동사 => 동작 => 함수 => 메소드
  •  객체의 동작을 조작하기 위한 도구.
  •  메서드(method) = '이름:함수' 의 쌍 객체에서 사용하는

기본타입 / 참조타입

  • 값을 저장하는 방식에 따라
  • 기본 타입 : 실제값이 저장. stack에 생성 / ===
  • 참조 타입 : 주소값이 저장. heap에 생성 / 바로가기 : 원본은 다른곳에 만들어 진다.
  • 기본형의 문제 발생하는 경우
  • 객체비교 : 어디에 저장하느냐에 따라서 기초/참조 객체 비교
  • 기본타입 : ===
  • 참조타입 : 비교하지 않음 ==인 경우에는 주소값이 같은지만 비교하기때문에 소용이 없음.
  • 매개변수

String

  • 문자열추출: substr()
  • 문자열치환: replace()
  • 문자열분할: split()
  • 문자열길이: length
  • 문자일결합: +

Prototype

  • 프로토타입 : 자신의 부모 역할을 하는 객체
  • 자기 자신을 생성하기 위해 사용된 객체 원형
  • 자식 객체는 부모 객체가 가진 프로퍼티나 메서드를 상속 받아 호출
  • 자바스크립트의 모든 객체는 Object 에서 파생 되었다.
__proto__ : 부모찾아가려할때 사용
  • 링크를 따라가면서 프로퍼티나 메서드를 찾음
  • Object 객체까지 존재하지 않으면 undefined 을 리턴

프로퍼티 찾기 순서

  1. 인스턴스에서 찾기
  2. 프로토타입 체인에서 찾기
constructor : 생성자를 이용하여 상속을 구현하고자할 때 사용
